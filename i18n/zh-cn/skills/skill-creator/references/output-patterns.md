# 输出模式

当技能需要产生一致且高质量的输出时，请使用这些模式。

## 模板模式

为输出格式提供模板。根据您的需求匹配严格程度。

**对于严格要求（如 API 响应或数据格式）：**

```markdown
## 报告结构

始终使用此确切的模板结构：

# [分析标题]

## 执行摘要
[关键发现的一段概述]

## 关键发现
- 带有支持数据的发现 1
- 带有支持数据的发现 2
- 带有支持数据的发现 3

## 建议
1. 具体可执行的建议
2. 具体可执行的建议
```

**对于灵活指导（当适应性有用时）：**

```markdown
## 报告结构

这是一个合理的默认格式，但请根据您的最佳判断使用：

# [分析标题]

## 执行摘要
[概述]

## 关键发现
[根据您发现的内容调整各部分]

## 建议
[根据特定上下文定制]

根据特定分析类型调整各部分。
```

## 示例模式

对于输出质量取决于看到示例的技能，提供输入/输出对：

```markdown
## 提交信息格式

根据以下示例生成提交信息：

**示例 1：**
输入：添加了使用 JWT 令牌的用户身份验证
输出：
```
feat(auth): 实现基于 JWT 的身份验证

添加登录端点和令牌验证中间件
```

**示例 2：**
输入：修复了报告中日期显示不正确的问题
输出：
```
fix(reports): 修正时区转换中的日期格式

在报告生成过程中一致地使用 UTC 时间戳
```

遵循此风格：type(scope): 简要描述，然后是详细解释。
```

示例帮助 Claude 更清楚地理解所需的风格和详细程度，比单独的描述更有帮助。
